Monthly Update: Nine Terminals From My Phone

How I replaced a $300/week cloud agent with $200/month Claude Code, a phone terminal, and a constitution file.

A month ago, I wrote about how I turned Claude Code into a 24/7 personal assistant. This is the follow-up. The architecture has changed significantly.

1. OpenClaw’s Priority Dropped

In the previous post, I was running OpenClaw (formerly Moltbot) on a cloud server — a Telegram-connected agent that handled daily tasks. It worked, but the economics were brutal: $230–300 in five or six days of token spend, plus constant babysitting from my local Claude Code to keep it stable.

I haven’t abandoned it entirely — it still handles some local company file processing reliably — but its priority has dropped dramatically. All heavy work now runs on Claude Code. The replacement for daily operations: an open-source project called Happy Coder. It gives me full terminal access from my phone — far more freedom than OpenClaw ever offered. Development, research, decision-making — I do essentially everything from my phone now.

My typical setup: 6 terminals in Docker, 3 on the local machine. Nine terminals total, all managed from my phone. Everything runs locally. No cloud servers.

2. What I Built

I’ve replicated about 90% of OpenClaw’s functionality natively in Claude Code:

  • Loop execution — Cron jobs + a custom Heartbeat system that triggers tasks on schedule
  • To-do list — job queue with priority management
  • Scheduled tasks — periodic bookmarks sync, research sweeps, environment checks
  • Notifications — task completion pings via Telegram bot or email, then I follow up through Happy Coder

The freedom is significantly higher than OpenClaw. And the cost is fixed: $200/month Max Plan. Done.

3. Why Not Claude Remote Control?

Anthropic recently launched Remote Control for Claude Code. One fatal flaw: sessions disappear when idle. You can’t leave it running for days.

With Happy Coder, I go days without opening my laptop. The machine just sits there like a Mac mini, running tasks autonomously. That level of persistence is non-negotiable for my workflow.

4. Environment Management

Two Claude Code instances in Docker and local, with Happy Coder managing both environments simultaneously:

  • Local CC manages Docker and all environment configs
  • Docker CC manages all websites and almost every development project

This covers 90% of what I need. OpenClaw is now both expensive and redundant.

5. The Research System

I’m building and testing a two-dimensional research system:

Bottom-up: Ultra Deep Research. The 260+ source intelligence system I wrote about before. It scrapes globally, denoises mathematically, and surfaces information asymmetries. This is the broad antenna.

Top-down: Deep Investigation. A new system that does targeted deep dives into a single company — contracts, personnel, financials, supply chain. A top-down search system.

With these two search systems combined, information acquisition becomes remarkably easy. Add the advisory council for decision-making, and my decision quality seems to have improved significantly. Decisions need execution, so I’m building a full loop. I’m calling it Cortex. This month my token efficiency is much higher — all costs stay within the $200 Max Plan.

6. The Constitution File

Some lessons on usage. CLAUDE.md might be the only file you ever need to maintain. I haven’t looked at code in a long time — well over a month. I just maintain the rules inside CLAUDE.md, and the “rules about rules.”

These can be called “meta-rules” — very abstract logic. For example, the rule for how to modify .md rules. It’s the single most important file, and you should maintain it with great care. Beyond that, let go of as much as you can. Don’t over-intervene.

As CLAUDE.md grows longer, you need to split it: move infrequently-used content into separate .md files and leave a pointer back to the main CLAUDE.md. I wrote these splitting rules into the file itself, so it handles its own maintenance. It’s like a constitution. Write it well, and the system runs itself.

After using this for so long, I’ve found the real bottleneck is the human brain. The amount of information a person can process each day is severely limited. So the goal is to minimize the number of decisions you need to make. How? Increase Claude’s autonomous development capability:

  • Plug in EvoMap — when it hits a hard problem, it finds methodologies on its own
  • Install a skill-finding skill — it discovers and installs high-quality capabilities autonomously
  • Periodically ingest my Twitter bookmarks via Heartbeat — a curated collection of methodologies that it absorbs on schedule

When it hits development difficulties, it can also invoke these on demand. The result: extremely high solo combat capability.

7. Ideation Is the Real Work

The most time-consuming part is ideation. I chat with it about all kinds of details, let it ask me questions, let it ideate with me. As long as the plan is solid and the logic is coherent, the probability of a one-shot implementation is surprisingly high.

So those flashy coding techniques actually matter less than truly thinking through what you want to build. Like the context management tricks I mentioned a month ago — I gotta say, the world evolves terrifyingly fast.

Opus 4.6 doesn’t need those tricks anymore. Its native context handling is already good enough. On this dimension, AI is evolving at a pace that’s hard to overstate.

8. Prompt Injection: Don’t Let Your Guard Down

There are more defenses against prompt injection now. But stories of stolen prompts and drained wallets are still real — you can’t afford to be complacent. The incidents have visibly decreased, with only scattered cases — or maybe most people who got burned just don’t talk about it.

Every major platform has opened agent-access channels. Wallets have added agent functionality. Moltbook’s growth hasn’t met my expectations — too much noise, not enough signal — but the overall agent ecosystem is slowly getting better.

Stay observant. Stay curious.

月度更新：手机上的九个终端

用 $200/月 的 Claude Code + 手机终端 + 一份宪法文件，替代了 $300/周的云端 Agent。

一个月前，我写了如何把 Claude Code 训练成 24/7 个人助理。这是后续更新。架构已经大幅改变。

1. OpenClaw 的优先级下降了

上篇文章里，我在云服务器上跑 OpenClaw（前身 Moltbot）—— 一个连接 Telegram 的 Agent，处理日常任务。能用，但经济账算不过来：五六天烧掉 $230–300 的 token，而且稳定性一般，经常需要本地的 Claude Code 去救火。

我没有完全放弃它 —— 一些本地公司文件处理还是很稳定的 —— 但优先级大幅下降了。所有重活现在都在 Claude Code 上跑。取而代之的是一个叫 Happy Coder 的开源项目。它让我在手机上拥有完整的终端控制权 —— 自由度比 OpenClaw 高太多。开发、研究、决策，我现在基本全在手机上完成。

日常配置：Docker 里 6 个终端，本地 3 个。一共 9 个终端，全部通过手机管理。所有文件和进程都跑在本地，不再依赖云服务器。

2. 核心功能实现

我已经在 Claude Code 上原生实现了 OpenClaw 约 90% 的功能：

  • 循环执行 —— Cron 任务 + 自制 Heartbeat 系统，定时触发
  • 待办列表 —— 带优先级的任务队列
  • 定时功能 —— 书签同步、研究扫描、环境检查，周期性运行
  • 通讯功能 —— 任务完成后 Telegram 或邮件通知，我再通过 Happy Coder 跟进

自由度显著高于 OpenClaw，且成本固定：$200/月 Max Plan。完事。

3. 为什么不用 Claude Remote Control？

虽然 Anthropic 最近发布了 Remote Control，但它有一个致命缺陷：Session 闲置后会自动消失。没法让它持续跑几天。

用 Happy Coder，我可以好几天不打开电脑干预。机器就像一台 Mac mini 一样稳定地自己跑着。这种持久性对我的工作流来说是刚需。

4. 环境管理

我的 Docker 和本地分别有两套 CC，Happy Coder 可以同时管理这两个环境：

  • 本地 CC 负责管理 Docker 和所有的环境配置
  • Docker CC 负责管理我的所有网站和几乎所有的开发项目

这套实践已经解决了 90% 的需求。OpenClaw 现在对我来说既贵又鸡肋。

5. 研究系统

目前我正在开发和测试我的研究系统。它包含两个维度：

自下而上：Ultra Deep Research。之前博客里提到的 260+ 源情报系统。全球采集、数学去噪、信息不对称探测。这是广角天线。

自上而下：Deep Investigation。一套新系统，能深入调查一家公司的合同等各类信息，是一个自上而下的搜索系统。

有了这两套搜索系统的结合，信息获取变得非常容易。再加上之前做的顾问团帮我做决策，我的决策质量似乎显著提高了。有了决策就要执行，所以我正在搭建一整套大循环 loop。这整套系统正在搭建中，我暂且称它为 Cortex。这个月我的 Token 使用效率比之前高很多，所有的成本基本上控制在 $200 Max Plan 以内。

6. 宪法文件

关于使用，我有一些心得。CLAUDE.md 文件可能是你唯一需要维护的文件。我已经很长时间 —— 不止一个月 —— 没有看过代码了。我只是维护 CLAUDE.md 里面所有的规则，以及「规则的规则」。

这些规则可以称之为「元规则」，是非常抽象的逻辑。比方说，修改 .md 规则的规则。它就是你需要的最重要的一个文件，你需要非常重视地维护它。除此之外，能放手则放手，不用做太多的干预。

CLAUDE.md 会随着使用变得越来越长，你可能需要做一些拆分：把不常用的内容外挂到别的 .md 文件里，留一个指针回到主要的 CLAUDE.md 即可。这些规则我也都写在了那个 .md 文件里面，所以它基本上都会自己处理。这就像宪法一样，只要你宪法写得好，这个系统就可以自己运行。

我发现用了这么久，其实真正的瓶颈是人的脑子。人每天脑子能处理的信息非常有限。所以要尽可能地降低你决策的次数。通过什么来实现呢？增强 Claude 的连续开发能力：

  • 给它外接 EvoMap，让它遇到难题自己去找方法论
  • 安装能找到高质量 Skills 的 Skill —— 自主发现和安装高质量能力
  • 自动读取我 Twitter 书签的 Heartbeat —— 里面是我沉淀的方法论集合，它会根据设置周期性地吸收

遇到开发困难的时候也可以自己去调用。所以现在 CC 的单兵作战能力是极强的。

7. Ideation 才是真正的工作

最花时间的地方在 ideation 阶段。我就是跟它聊聊各种各样的细节，让它去问我，让它跟我一起去 ideate。其实只要 plan 做得好，整个项目里的逻辑是通顺的，它 one-shot 开发完的概率还是蛮高的。

所以那些比较炫酷的编程技巧，其实不如去真的把你要做的事情想清楚。就像我一个月前所说的一些上下文管理小技巧，I gotta say 这个世界进化得太快了。

Opus 4.6 其实已经不需要我去用这样的技巧，它本身的上下文能力已经足够了。所以在这些层面，这些 AI 进化得简直飞快。

8. Prompt Injection：不能掉以轻心

至于 Prompt Injection，大家有各种各样的方式可以去防御了。网上说的那些因为 Prompt 被盗、把钱骗走之类的问题说实话还是存在的，所以还是不能掉以轻心。但是它可见地变少了，只有一些零星的案例 —— 也有可能是大部分人被骗了不愿意说吧。

各大厂都开启了 Agent 访问的各种通道，各种钱包也接了 Agent 的功能。虽然 Moltbook 的发展没有我预想的好，灌水很严重，但是整个 Agent 生态还是在慢慢变好的。

保持观察，保持好奇心。
